## Railway Variables
Обязательные переменные:

- `BOT_TOKEN` — токен @klicmediabot
- `ADMIN_ID` — Telegram ID владельца
- `DATABASE_URL` — строка подключения PostgreSQL

### Как подключить PostgreSQL в Railway
1. В проекте Railway нажмите **+ New** → **Database** → **PostgreSQL**.
2. Откройте сервис `klicmediabot` → **Variables**.
3. Добавьте `DATABASE_URL` как reference на `DATABASE_URL` из PostgreSQL-сервиса.
4. Убедитесь, что также есть `BOT_TOKEN` и `ADMIN_ID`.
5. После сохранения Railway сделает новый deploy.

Без `DATABASE_URL` бот запустится на SQLite только для теста, но такая база может исчезнуть после redeploy. Для рабочего КЛИК обязательно используйте PostgreSQL.

## Первый тест
1. Напишите боту `/start` с аккаунта владельца — должны появиться закреплённые кнопки админ-панели.
2. Нажмите `👤 Режим пользователя` и создайте профиль бизнеса.
3. Создайте заявку — она должна получить статус `pending` и прийти владельцу на модерацию.
4. Нажмите `✅ Одобрить` — после этого заявка станет активной и уйдёт к зарегистрированным креаторам.
5. Проверьте `📥 Выгрузить Excel`.


## v4 — реферальная система
- У каждого креатора и бизнеса есть кнопка «Пригласить».
- Персональная ссылка: `https://t.me/<bot>?start=ref_<telegram_id>`.
- Реферал закрепляется только при первом входе и не меняется повторным `/start`.
- Самореферал запрещён.
- В админке есть «🔗 Рефералы».
- Excel-выгрузка содержит отдельный лист «Рефералы».
- Данные хранятся в PostgreSQL и переживают новые деплои.
